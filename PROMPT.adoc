= AI-driven development and maintenance

An `agents.md` file is where you define all the specifics of a task: the agent’s persona, the exact tech stack it should know, the implementation file structure, workflows, and the explicit commands it can run.
It’s also where you provide examples and, most importantly, set clear boundaries of what not to do.

See 
https://github.blog/ai-and-ml/github-copilot/how-to-write-a-great-agents-md-lessons-from-over-2500-repositories[How to write a great agents.md]
for more information.

See
https://github.com/github/awesome-copilot[Awesome GitHub Copilot]
for a community created collection of custom agents and instructions.
