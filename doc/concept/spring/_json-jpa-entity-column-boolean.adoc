= Column for type `boolean`

== Default Implementation

.Entity
[source,java,options="nowrap"]
----
@Column(name = "flag") <1>
@Getter <2>
@JsonProperty <3>
private boolean flag; <4>
----

<1> The field is marked with `@Column`.
The column name in the database is explicitly specified.
<2> The field is marked with `@Getter`.
This allows the current value to be queried via a getter method.
<3> The field is marked with `@JsonProperty`.
The value is included in the JSON structure of the REST API.
<4> The field has the primitive data type `boolean`.
Using the primitive type avoids null issues.
This is a mutable field due to requirements from the JPA provider.

.Liquibase
[source,xml,options="nowrap"]
----
<column name="flag" type="BOOLEAN" defaultValueBoolean="false"> <1>
    <constraints nullable="false"/> <2>
</column>
----

<1> The column type is `BOOLEAN` with a default value of `false`.
<2> The column is marked as NOT NULL.

== Property initialization

`boolean` properties are not nullable and initialized in the constructor.

== Property manipulation

`boolean` properties are typically only changed through REST endpoints.

`boolean` properties are copied in the `withId` operation during entity cloning.

Add a setter if a controller must be able to update a property directly.

[source,java,options="nowrap"]
----
@JsonIgnore <1>
public Entity setFlag(final boolean flag) { <2>
    this.flag = flag;
    return this; <3>
}
----

<1> The method is marked with `@JsonIgnore` as it is not part of the JSON API.
<2> The method accepts a primitive `boolean` argument.
<3> The method returns `this` for operation chaining.

== Property verification

`boolean` properties participate in the `isEqual` method comparison.
Use `==` for direct comparison.

The `verify` operation may include custom validation logic.

== Property serialization

`boolean` properties are by default part of the JSON structure.

== Tests

Create test data from JSON using `fromJson` method of the entity class.

.Example from Visit entity
[source,java,options="nowrap"]
----
Visit createWithText(final String text) {
    return Visit.fromJson("""
        {
            "date":"2021-04-22",
            "text":"%s"
        }
        """.formatted(text))
        .setPet(Pet.fromJson(...))
        .setVet(Vet.fromJson(...))
}
----

The `equalsHashcodeToString` test verifies equality operations.

The `withId` test verifies cloning.

The `json{Name}` parameterized test verifies the happy path, e.g.

- `verify` does not throw an exception
- correct deserialization from JSON
- assert property value

.Example from Visit entity
[source,java,options="nowrap"]
----
@ParameterizedTest
@ValueSource(booleans =  {true, false})
void jsonBillable(final boolean billable) {
    final var value = Visit.fromJson("""
        {
            "billable":"%b"
        }
        """.formatted(billable));
    assertDoesNotThrow(value::verify);
    assertEquals(billable, value.isBillable());
}
----

The `json{Name}Constraints` parameterized test verifies constraints, e.g.

- `verify` throws an exception
- invalid values are rejected
- missing values are rejected

.Example from Vet entity
[source,java,options="nowrap"]
----
@ParameterizedTest
@ValueSource(strings = {
        "JA",
        "NEIN",
})
void jsonBillableConstraints(final String text) {
    final var json = """
        {
            "billable":"%s"
        }
        """.formatted(text);
    assertThrows(IllegalArgumentException.class, () -> Visit.fromJson(json).verify());
}
----
