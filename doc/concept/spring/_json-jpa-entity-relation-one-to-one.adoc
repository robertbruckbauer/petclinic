= Relation of kind `OneToOne`

== Domain story

This implementation guide applies whenever exactly one entity instance is directly associated with exactly one other entity instance, and the foreign key is owned by the referencing side.
The referenced entity is read lazily.

.Example
....
A visit can be linked to exactly one `Invoice` that records the billing details for that appointment.
The foreign key `invoice_id` is stored on the `visit` table.
The owner is read lazily.
As the recipient of the invoice the owner is exposed as a lightweight item object in API responses to avoid circular references.
....

== Default implementation

.Property - Required Relationship
[source,java,options="nowrap"]
----
@OneToOne(
        fetch = FetchType.LAZY, <1>
        optional = false) <2>
@JoinColumn(
        name = "invoice_id", 
        referencedColumnName = "id") <3>
@Getter <4>
@JsonProperty(access = JsonProperty.Access.WRITE_ONLY) <5>
private Invoice invoice; <6>
----

<1> The field is marked with `@OneToOne` with `LAZY` fetch type.
The parent entity is not loaded immediately; it's fetched only when accessed.
<2> The `optional = false` indicates this relationship is required (not nullable).
<3> The `@JoinColumn` annotation specifies the foreign key column name and the referenced id column name in the parent table.
It is the clean way to provide the parent field and avoid a mapping table in case of an uni-directional mapping. 
<4> The field is marked with `@Getter`.
This allows the parent entity to be accessed via a getter method.
<5> The field is marked with `@JsonProperty`.
The full parent object is only used during deserialization, not in JSON responses.
<6> The field has the entity type of the parent.
This is a mutable field due to requirements from the JPA provider.
It can be null during initialization but must be set before any persistence operation.

== Relation initialization

The relation is typically initialized to `null` and set later through setter methods.
`null` represent the absence of a relationship.

The `withId` method must copy the relation so that cloned instances remain semantically equal.

== Relation manipulation

.Setter
[source,java,options="nowrap"]
----
@JsonIgnore <1>
public Visit setInvoice(@NonNull final Invoice invoice) { <2>
    this.invoice = invoice;
    return this; <3>
}
----

<1> The method is marked with `@JsonIgnore` as it is not part of the JSON API.
<2> The method accepts the parent entity.
Remove `@NonNull` for an optional relation.
<3> The method returns `this` for method chaining.

== Relation verificationation

The relation participates in the `isEqual` method comparison.
Use `Objects#equals`.

Do not use relations in the `verify` operation.

== Relation serialization

The relation is by default *not* part of the JSON structure.

The relation can be added to the JSON structure only with item classes.
This prevents circular references and reduces payload size.
Be aware of the potential impact on performance.

.Item
[source,java,options="nowrap"]
----
@Override <1>
protected void extraJson(@NonNull final Map<String, Object> allExtra) {
    allExtra.put("ownerItem", OwnerItem.fromValue(visit)); <2>
}
----

<1> Callback from the base operation with `@JsonAnyGetter` annotation.
<2> The Owner entity is converted to a simple state transfer object.
This does break lazy loading, but it makes sense because the recipient is always needed.

== `<Entity>Test` approach

Create test data from JSON using `fromJson` method of entity classes.

== `<Entity>RepositoryTest` approach

A `save<Entity>` test verifies the happy path for a persistence operation.
Persist fixtures parameterized with the unique keys needed to provide referenced required entities with a fixed primary key value.

.Example
[source,text,options="nowrap"]
----
@ParameterizedTest
@ValueSource(strings = {
        "Lorem ipsum."
})
void saveInvoice(final String text) {
    final var value0 = createWithText(text);
    assertFalse(value0.isPersisted());
    assertEquals(0L, value0.getVersion());
    assertNotNull(value0.getId());
    assertNull(value0.getVisit());
    // other assertions

    final var visit = saveVisit("Ipso facto");
    final var value1 = invoiceRepository.save(value0.setVisit(visit));
    assertNotNull(value1);
    assertNotSame(value0, value1);
    assertTrue(value1.isPersisted());
    assertEquals(0L, value1.getVersion());
    assertNotNull(value1.getId());
    assertNotNull(value1.getVisit());
    assertTrue(value1.isEqual(value0));
}
----

A `save<Entity>UniqueContraint` verifies the unique constraint.

.Example
[source,text,options="nowrap"]
----
@Test
void saveInvoiceUniqueConstraint() {
    final var visit = saveVisit("Ipso falso");
    final var value1 = invoiceRepository.save(createWithText("Invoice one").setVisit(visit));
    assertNotNull(value1);
    assertThrows(DataIntegrityViolationException.class, () -> {
        final var value2 = invoiceRepository.save(createWithText("Invoice two").setVisit(visit));
        assertNotNull(value2);
        invoiceRepository.flush();
    });
}
----

== `<Entity>GraphqlTest` approach

A `queryAll<Entity>` test verifies the happy path for a query operation.
Create fixtures parameterized with the unique keys needed to provide referenced required entities with a fixed primary key value.

.Example
[source,text,options="nowrap"]
----
@Test
void queryAllInvoice() {
    final var value = createWithText("Ipso facto.");
    when(invoiceRepository.findAll())
            .thenReturn(List.of(value));
    when(visitRepository.findAllById(any()))
            .thenReturn(List.of(value.getVisit()));
    final var data = graphQlTester
            .document("{allInvoice{text visit{id}}}")
            .execute();
    assertNotNull(data);
    data.path("allInvoice[0].text")
            .hasValue()
            .entity(String.class)
            .isEqualTo(value.getText());
    data.path("allInvoice[0].visit.id")
            .hasValue()
            .entity(UUID.class)
            .isEqualTo(value.getVisit().getId());
    verify(invoiceRepository).findAll();
    verifyNoMoreInteractions(invoiceRepository);
    verify(visitRepository).findAllById(any());
    verifyNoMoreInteractions(visitRepository);
}
----
