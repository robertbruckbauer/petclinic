= Relation of kind `ManyToOne`

== Domain story

This implementation guide applies whenever many instances of one entity are associated with exactly one instance of another entity, and the foreign key is owned by the referencing (child) side.

.Example
....
A pet can belong to exactly one owner.
The foreign key `owner_id` is stored on the `pet` table.
The owner is read lazily.
As the person responsible for the owner is exposed as a lightweight item object in API responses to avoid circular references.
....

== Default implementation

.Property - Required Relationship
[source,java,options="nowrap"]
----
@ManyToOne(
        fetch = FetchType.LAZY, <1>
        optional = false <2>
)
@JoinColumn(
        name = "owner_id", 
        referencedColumnName = "id") <3>
@Getter <4>
@JsonProperty(access = JsonProperty.Access.WRITE_ONLY) <5>
private Owner owner; <6>
----

<1> The field is marked with `@ManyToOne` with `LAZY` fetch type.
The parent entity is not loaded immediately; it's fetched only when accessed.
<2> The `optional = false` indicates this relationship is required (not nullable).
<3> The `@JoinColumn` annotation specifies the foreign key column name and the referenced id column name in the parent table.
It is the clean way to provide the parent field and avoid a mapping table in case of an uni-directional mapping. 
<4> The field is marked with `@Getter`.
This allows the parent entity to be accessed via a getter method.
<5> The field is marked with `@JsonProperty`.
The full parent object is only used during deserialization, not in JSON responses.
<6> The field has the entity type of the parent (`Owner`).
This is a mutable field due to requirements from the JPA provider.
It can be null during initialization but must be set before any persistence operation.

.Property - Optional Relationship
[source,java,options="nowrap"]
----
@ManyToOne(
        fetch = FetchType.LAZY, <1>
        optional = true) <2>
@JoinColumn(
        name = "owner_id", 
        referencedColumnName = "id") <3>
@Getter <4>
@JsonProperty(access = JsonProperty.Access.WRITE_ONLY) <5>
private Owner owner; <6>
----

<1> The field is marked with `@ManyToOne` with `LAZY` fetch type.
The entity is not loaded immediately; it's fetched only when accessed.
<2> The `optional = true` indicates this relationship is optional (nullable).
<3> The `@JoinColumn` annotation specifies the foreign key column name and the referenced id column name in the parent table.
It is the clean way to provide the parent field and avoid a mapping table in case of an uni-directional mapping. 
<4> The field is marked with `@Getter`.
This allows the parent entity to be accessed via a getter method.
<5> The field is marked with `@JsonProperty`.
The full parent object is only used during deserialization, not in JSON responses.
<6> The field has the entity type of the parent (`Owner`).
This is a mutable field due to requirements from the JPA provider.
It can remain null, representing the absence of a relationship.

== Liquibase changeset

.Column
[source,xml,options="nowrap"]
----
<addColumn tableName="pet"> <1>
    <column name="owner_id" type="UUID"> <2>
        <constraints nullable="false"/> <3>
    </column>
</addColumn>
----

<1> The existing table is updated for the relation.
<2> The foreign key column is defined in the target table.
<3> The `nullable="false"` constraint enforces required relationships.
Use `nullable="true"` constraint for optional relationships.

.Constraint
[source,xml,options="nowrap"]
----
<addForeignKeyConstraint
        constraintName="fk_pet_owner_id" <1>
        baseTableName="pet" <2>
        baseColumnNames="owner_id"
        referencedTableName="owner" <3>
        referencedColumnNames="id"
/>
----

<1> A named constraint makes it easier to identify in database tools.
<2> The `baseTableName` is the child table containing the foreign key.
<3> The `referencedTableName` is the parent table being referenced.

== Relation initialization

The relation is typically initialized to `null` and set later through setter methods.
`null` represent the absence of a relationship.

The `withId` method must copy the relation so that cloned instances remain semantically equal.

== Relation manipulation

.Setter
[source,java,options="nowrap"]
----
@JsonIgnore <1>
public Pet setOwner(@NonNull final Owner owner) { <2>
    this.owner = owner;
    return this; <3>
}
----

<1> The method is marked with `@JsonIgnore` as it is not part of the JSON API.
<2> The method accepts the parent entity.
Remove `@NonNull` for an optional relation.
<3> The method returns `this` for method chaining.

== Relation verification

The relation participates in the `isEqual` method comparison.
Use `Objects#equals`.

Do not use relations in the `verify` operation.

== Relation serialization

The relation is by default *not* part of the JSON structure.

The relation can be added to the JSON structure only with item classes.
This prevents circular references and reduces payload size.
Be aware of the potential impact on performance.

.Item
[source,java,options="nowrap"]
----
@Override <1>
protected void extraJson(@NonNull final Map<String, Object> allExtra) {
    allExtra.put("ownerItem", OwnerItem.fromValue(owner)); <2>
}
----

<1> Callback from the base operation with `@JsonAnyGetter` annotation.
<2> The parent entity is converted to a simple state transfer object.

== `<Entity>Test` approach

Create test data from JSON using `fromJson` method of entity classes.
Create fixtures parameterized with the unique keys needed to provide referenced required entities with a fixed primary key value.

The `equalsHashcodeToString` test verifies equality operations.
For `isEqual` to succeed, referenced entities must have a fixed primary key.

The `withId` test verifies cloning.

The `writeJson` test verifies serialisation to JSON.
relations must always be absent from the JSON structure.

== `<Entity>RepositoryTest` approach

A `save<Entity>` test verifies the happy path for a persistence operation.
Persist fixtures parameterized with the unique keys needed to provide referenced required entities with a fixed primary key value.

.Example
[source,text,options="nowrap"]
----
@ParameterizedTest
@ValueSource(strings = {
        "Käthe",
        "Jörg",
        "Ümit"
})
void savePet(final String name) {
    final var value0 = createWithName(name);
    assertFalse(value0.isPersisted());
    assertEquals(0L, value0.getVersion());
    assertNotNull(value0.getId());
    assertNull(value0.getOwner());
    // other assertions

    final var owner = saveOwner("Max Mustermann");
    final var value1 = petRepository.save(value0.setOwner(owner));
    assertNotNull(value1);
    assertNotSame(value0, value1);
    assertTrue(value1.isPersisted());
    assertEquals(0L, value1.getVersion());
    assertNotNull(value1.getOwner());
    assertTrue(value1.isEqual(value0));
}
----

== `<Entity>GraphqlTest` approach

A `queryAll<Entity>` test verifies the happy path for a query operation.
Create fixtures parameterized with the unique keys needed to provide referenced required entities with a fixed primary key value.

.Example
[source,text,options="nowrap"]
----
@Test
void queryAllPet() {
    final var owner = Owner.fromJson("""
            {
                "name":"Alice Cooper"
            }
            """);
    final var value = createWithName("Tom")
            .setOwner(owner);
    when(petRepository.findAll())
            .thenReturn(List.of(value));
    when(ownerRepository.findAllById(anySet()))
            .thenReturn(List.of(owner));
    final var data = graphQlTester
            .document("""
                    {allPet{name owner{name}}}
                    """)
            .execute();
    assertNotNull(data);
    data.path("allPet[0].name")
            .hasValue()
            .entity(String.class)
            .isEqualTo(value.getName());
    data.path("allPet[0].owner.name")
            .hasValue()
            .entity(String.class)
            .isEqualTo(owner.getName());
    verify(petRepository).findAll();
    verifyNoMoreInteractions(petRepository);
    verify(ownerRepository).findAllById(anySet());
    verifyNoMoreInteractions(ownerRepository);
}
----
